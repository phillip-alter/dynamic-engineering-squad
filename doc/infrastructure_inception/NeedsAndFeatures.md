# Milestone \#2

## List of Needs

1. Need \- Ability to take pictures from the app or upload pictures  
2. Need \- Identify pictures for damage cost analysis  
3. Need \- Information on who to connect with in city planning  
4. Need \- Email City Planning 
5. Need \- Geolocate issue to be identified on a map
6. Need \- User registration, email/password
7. Need \- User authentication, so user can log in
8. Need \- Ability to flag images/accounts
9. Need \- Ability to flag/report/ban users for inappropriate names and pictures abuse
10. Need \- Ability for users to earn a reputation based on how credible their posts are
11. Need \- User Roles (user/moderator/admin)


## List of Features (with priority High/Medium/Low)

### Core Reporting & Discovery
1. Submitting an infrastructure issue (photo, location, description) — High  
2. Feed of latest reported infrastructure issues — Medium  
3. Maps screen showing infrastructure issues near the user — Medium  
4. Connect road cameras to view roads/potholes in real time — Low  

### Community & Validation
5. Upvote / verification system for existing issues — High  
6. Naming system for reported infrastructure issues — High  
7. Celebration or visual indicator when an infrastructure issue is resolved — Low  
8. Flagging system for inappropriate or false posts — Medium  

### User Accounts & Roles
9. User accounts with authentication — High  
10. User roles (e.g., reporter, moderator, administrator) — High  
11. User dashboard showing activity and contributions — Low  

### Gamification & Social
12. Leaderboard of users with highest contribution points — Medium  
13. Leaderboard of worst infrastructure damage photos — Low  
14. Sharing infrastructure issues to social media — Low  

## What can a user do with it?

1. Upload pictures of identified infrastructure issues to participate in the ‘game’  
2. Get an estimated cost and/or time of repair for the issue  
3. As issues gain popularity, email the city planning or infrastructure repair group about the issue  
4. API issues out email to city planning/infrastructure repair team  
5. Pin location of infrastructure issue  
6. Get the user involved in getting the highest damage cost infrastructure issue  
7. Get the user involved in getting the highest amount of reported infrastructure issues  
8. User can share to social media to show others their report  
9. Allow users to vote on names for infrastructure issues  
10. Allow users to upvote and “verify” that infrastructure issue still exists  
11. Users celebrate when infrastructure issue is resolved  
12. Allow users to observe and view issues in real time  
13. Users can guess location of issue  
    

## What will it do for a user?

1. It will embolden a citizen to do their civic duty by reporting bad infrastructure.  
2. Show all the different pics of infrastructure damage to the city they live in.  
3. Help them become more involved in their city by holding city officials accountable.  
4. Encourage users to come back and continue to report issues  
5. “Pokemon Go” for Infrastructure \- gotta catch em all

## What problems will it solve?

1. Encouraging community members to get involved with infrastructure  
2. Hopefully infrastructure will be fixed or at least acknowledged by the city involved  
3. Allow regular citizens to feel good about their city

## What needs do your users have?

1. A need to have clean, working infrastructure in a city you live in that you pay taxes in.  
2. The need to feel they are participating in their community/community involvement  
3. They need the ability to access our app  
4. They need clear instructions on how to use the app  
5. A clean user interface that is simple to understand and allows them to quickly add issues without fluff or issue

## Are there needs for the system?

1. The system must validate uploaded images  
2. The system must validate user names  
3. The system must handle failed or unavailable external APIs  
4. The system with clean UI/UX to make sure the reporting process is fast, easy, and enjoyable, not slogging through forms.   
5. The system must prevent duplicate infrastructure issue reports for the same location. 
6. The system must be able to flag/report/ban users for inappropriate names or pictures.

### Potential User Stories

- User Registration (4 pts): As a new user, I would like to create a new account with a username, email, and password so I can start submitting reports and earning points.
- User Login (2 pts): As a registered user, I would like to log in using my credentials so I can access my account and private features.
- Admin: Ban/Suspend User (4 pts): As an Administrator, I would like to ban a user who abuses the system so I can maintain a safe and clean community.
- Admin: Manage User Roles (4 pts): As an Administrator, I would like to promote a standard user to a "Moderator" so I can share the workload of verifying infrastructure reports.
- Route Protection (2 pts): As the Product Owner, I would like to block access to specific pages for non-logged-in users so I can prevent spam and unauthorized submissions.
- User Dashboard (2 pts): As a competitive user, I would like to view my personal dashboard so I can see my total contribution points and reputation score.
- Edit User Profile (2 pts): As a user, I would like to update my email address or bio so I can keep my contact info current and express my identity.
- Upload Profile Picture (2 pts): As a user, I would like to upload a custom avatar image so I can be recognized on the leaderboard.
- Secure Logout (1 pt): As a user, I would like to log out of the application so I can keep my account safe when I leave a shared computer.
- View Public Profile (1 pt): As a user, I would like to click on another user's name in the feed so I can see their public stats and avatar.
